Smart Cache Manual
Chapter 17 - Future

Smart Cache is now nearly finished and I hope, that there are no fatal bugs left. If you want to add some feature see also Sponsoring Smart Cache development, Section 6.1.

Next versions will have:

  • (maybe) pipelining, if tests shows better network performance. Anybody wants to contribute some results?
  • (maybe) Internal support for FTP protocol. Parsing of FTP server LIST replies is a small problem.
  • (maybe) support for multiple parent proxies. Who really needs this?
  • (not worth of programming effort) auto support for HTTP-resume on aborted download; aborted downloads are now deleted. Web pages are quite short < 100K, so it does not make sense. For bigger or batch downloads use special programs which handles this better; for example, my Download Machine or wget.
  • (never?) Auto run Garbage collection.
  • (never) AWT/SWING GUI. Slows down things a lot. Smart Cache must not depends on graphics engine (Presentation Manager, X11, ..).
  • (never) non Open Source version.
  • (never) things which will require to do HTML parsing on data streams.
